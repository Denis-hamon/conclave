The organizational layer for Claude Managed Agents.

Conclave is an open-source framework that brings roles, deliberation, and auditability to multi-agent systems built on Anthropic primitives — Managed Agents, MCP, and the Claude model family.

A bureau, not a graph.

Most multi-agent frameworks model tasks or graphs. Real organizations don't work that way. They work through roles that hold persistent accountability, deliberate together, and leave an audit trail. Conclave makes that the primitive.

01

Roles, not tasks.

Define your org in YAML — CPO, TechLead, QA — each with a persona, reporting line, tools, and persistent memory. A role is a first-class object, not an ad-hoc function.

02

Token-aware routing.

A classifier decides per task: repetitive work goes to a Haiku self-correction loop, novel work goes to Sonnet. Typical savings: 60–80% versus running everything on a frontier model.

03

Decision Trail, always on.

Every inter-agent message is written to a JSONL trail with provenance, reasoning, and cost. Replayable, diffable, postmortem-ready — no extra wiring.

Define an organization. Give it a goal.

A Conclave is a small group of Claude-backed agents, each anchored to one role, working through a structured deliberation until a concrete output is produced. The file below is the entire configuration.

examples/product_squad.yml 
org:
  name: "Product Squad"
  deliberation: hierarchy
  backend: managed_agents   # anthropic | managed_agents

  agents:
    - role: CPO
      persona: "Strategic. Defines scope. Validates business value."
      tools: [notion, slack]
      force_model: claude-sonnet-4-6

    - role: TechLead
      persona: "Tight specs. Flags blockers. Offers two options, not one."
      reports_to: CPO
      tools: [github, linear]

    - role: QA_Engineer
      persona: "Adversarial. Never approves without a test plan."
      reports_to: TechLead
      tools: [github, browserbase]

Built on Anthropic primitives, by design.

Conclave does not reinvent the agent. It wraps Managed Agents sessions in a coordination layer, binds tool access to MCP servers, and routes across the Claude model family — so every future Anthropic release is a step forward, not a rewrite.

Where it sits next to existing tools.

Multi-agent frameworks have proliferated. Most rebuild primitives Anthropic already ships. Conclave chooses the opposite: stay thin, lean on Managed Agents and MCP, and add the one thing nobody else does — the organizational layer.

Forward-compatible by design.

Conclave's backend abstraction means switching from the stateless messages.create() path to native Managed Agents sessions is a one-line YAML change. When the beta goes GA, the migration is a no-op. 

org:
  backend: managed_agents
